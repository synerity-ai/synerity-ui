How to Use RadioButton

Basic Implementation

Radio buttons are used when you need to select one option from a group of mutually exclusive options.

Key Features

  • Mutual Exclusivity: Only one option can be selected at a time within a group
  • Accessibility: Proper labeling and keyboard navigation support
  • States: Support for normal, checked, disabled, and readonly states
  • Sizes: Small, medium, and large size variants
  • Styling: Customizable appearance with Tailwind CSS classes

Best Practices

  • Use radio buttons when users need to select exactly one option from a set
  • Group related radio buttons with the same name attribute
  • Provide clear, descriptive labels for each option
  • Use consistent sizing and spacing within a group
  • Consider using descriptions for complex options
  • Ensure proper keyboard navigation and screen reader support

RadioButton Documentation

Properties

Property Type Default Description
type string "radio" Input type attribute
name string - Name attribute for grouping radio buttons
value string - Value of the radio button
checked boolean false Whether the radio button is checked
disabled boolean false Whether the radio button is disabled
readonly boolean false Whether the radio button is readonly

Events

Event Type Description
change Event Fired when the radio button value changes
click Event Fired when the radio button is clicked

Accessibility

  • Use proper label elements associated with radio buttons
  • Ensure each radio button has a unique id attribute
  • Use the same name attribute for related radio buttons
  • Provide descriptive text for screen readers
  • Support keyboard navigation (Tab, Arrow keys, Space)
  • Use aria-describedby for additional context if needed
