How to Use Button

Installation

Import the Button component from the UI library:

import { Button } from 'ui-lib';

// In your component imports
imports: [Button]

Basic Usage

Use the Button component with different variants:

<sui-button 
  type="button"
  variant="primary"
  (click)="onClick()"
>Click Me</sui-button>

Component Properties

  • type: HTML button type (button, submit, reset)
  • variant: Button style variant (primary, secondary, outline, ghost, danger)
  • size: Button size (small, medium, large)
  • disabled: Disable the button
  • loading: Show loading state
  • fullWidth: Make button full width

Button Variants

  • primary: Main action button with solid background
  • secondary: Secondary action with muted styling
  • outline: Transparent background with border
  • ghost: Transparent background without border
  • danger: Destructive action with warning colors

Button Sizes

  • small: Compact button for dense layouts
  • medium: Standard button size (default)
  • large: Prominent button for important actions

Common Use Cases

  • Forms: Submit and reset buttons
  • Actions: Primary and secondary actions
  • Navigation: Call-to-action buttons
  • Dialogs: Confirm and cancel buttons
  • Cards: Action buttons within content

Best Practices

  • Clear Labels: Use descriptive button text
  • Consistent Sizing: Use appropriate sizes for context
  • Loading States: Show loading for async operations
  • Accessibility: Ensure proper focus and keyboard navigation
  • Color Usage: Use danger variant sparingly for destructive actions

Form Integration

Buttons work seamlessly with forms:

<form (ngSubmit)="onSubmit()">
  <!-- Form fields -->
  
  <div class="form-actions">
    <sui-button type="submit" variant="primary">
      Submit Form
    </sui-button>
    
    <sui-button type="button" variant="outline" (click)="onCancel()">
      Cancel
    </sui-button>
  </div>
</form>

API Documentation

Properties

Property Type Default Description
type string 'button' HTML button type
variant string 'primary' Button style variant
size string 'medium' Button size
disabled boolean false Disable the button
loading boolean false Show loading state
fullWidth boolean false Make button full width

Events

  • click: Emitted when the button is clicked

Variants

Variant Description Use Case
primary Solid background with primary color Main actions, form submissions
secondary Solid background with secondary color Secondary actions
outline Transparent with colored border Alternative actions
ghost Transparent with hover effects Subtle actions, links
danger Red background for destructive actions Delete, remove, destructive actions
