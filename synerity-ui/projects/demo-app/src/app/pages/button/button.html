Button

Interactive buttons with multiple variants, sizes, and states for user actions.

Basic Variants

Primary Secondary Danger Outline Ghost

Note: The sui-button component supports primary, secondary, danger, outline, and ghost variants. For custom colors like success (green) or warning (yellow), you can extend the component or use custom CSS classes. 

<!-- Primary Button -->
<sui-button variant="primary">Primary</sui-button>

<!-- Secondary Button -->
<sui-button variant="secondary">Secondary</sui-button>

<!-- Danger Button -->
<sui-button variant="danger">Danger</sui-button>

<!-- Outline Button -->
<sui-button variant="outline">Outline</sui-button>

<!-- Ghost Button -->
<sui-button variant="ghost">Ghost</sui-button>

Button Sizes

Small Medium Large 
<!-- Small Button -->
<sui-button variant="primary" size="sm">Small</sui-button>

<!-- Medium Button -->
<sui-button variant="primary" size="md">Medium</sui-button>

<!-- Large Button -->
<sui-button variant="primary" size="lg">Large</sui-button>

Installation

The Button component is part of the Synerity UI library. Install it using npm:

npm install @synerity/ui

Styling with Tailwind CSS

Buttons use Tailwind CSS classes for styling. Here are the key classes:

px-4 py-2 Horizontal and vertical padding
bg-blue-600 Background color
text-white Text color
rounded-md Border radius
hover:bg-blue-700 Hover state

Props

Prop Type Default Description
variant 'primary' | 'secondary' | 'outline' | 'ghost' | 'danger' 'primary' Visual style variant of the button
size 'sm' | 'md' | 'lg' 'md' Size of the button
type 'button' | 'submit' | 'reset' 'button' HTML button type attribute
disabled boolean false Whether the button is disabled
loading boolean false Whether the button is in loading state
fullWidth boolean false Whether the button should take full width of its container
icon string | null null Icon to display in the button (HTML string or icon name)
iconPosition 'left' | 'right' 'left' Position of the icon relative to the button text
