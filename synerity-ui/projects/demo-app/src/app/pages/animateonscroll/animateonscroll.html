Animate On Scroll

Component that triggers animations when elements come into view during scrolling, perfect for creating engaging user experiences.

Basic Animation

Scroll down to see the animation

Animated Card

This card will fade in and slide up when it comes into view. Scroll down to see more examples.

Interactive: Scroll down to see the animation trigger. The element animates when it enters the viewport. 

<sui-animate-on-scroll animation="fadeInUp" threshold="0.1" duration="1000">
  <div class="bg-blue-50 border border-blue-200 rounded-lg p-6">
    <h4 class="text-lg font-semibold text-blue-900 mb-2">Animated Card</h4>
    <p class="text-blue-700">This card will fade in and slide up when it comes into view.</p>
  </div>
</sui-animate-on-scroll>

Staggered: Multiple cards with different delays create a cascading animation effect. 

<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
  <sui-animate-on-scroll 
    *ngFor="let item of demoItems; let i = index"
    [animation]="i % 2 === 0 ? 'fadeInUp' : 'fadeInDown'"
    [delay]="i * 100"
    [duration]="800"
    [threshold]="0.1">
    <div class="bg-gradient-to-br from-blue-50 to-indigo-50 border border-blue-200 rounded-lg p-4">
      <h5 class="font-semibold text-blue-900 mb-2">Card Title</h5>
      <p class="text-sm text-blue-700">Card content goes here</p>
    </div>
  </sui-animate-on-scroll>
</div>

Installation

The Animate On Scroll component is part of the Synerity UI library. Install it using npm:

npm install @synerity/ui

Basic Usage

Import and use the Animate On Scroll component in your Angular application:

import { AnimateOnScroll } from '@synerity/ui';

@Component({
  selector: 'app-example',
  imports: [AnimateOnScroll],
  template: `
    <sui-animate-on-scroll animation="fadeInUp">
      <div class="my-content">Content to animate</div>
    </sui-animate-on-scroll>
  `
})
export class ExampleComponent {}

Props

Property Type Default Description
animation string 'fadeInUp' Animation type to apply when element comes into view
threshold number 0.1 Percentage of element that must be visible to trigger animation (0.0 to 1.0)
delay number 0 Delay in milliseconds before animation starts
duration number 1000 Duration of the animation in milliseconds
once boolean true Whether animation should only trigger once
