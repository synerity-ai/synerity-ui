SpeedDial Component

Floating action button with expandable menu items for quick access to common actions.

Basic SpeedDial

Click the speed dial button to expand and see available actions.

<sui-speed-dial 
  [items]="basicItems"
  buttonIcon="⚡"
  direction="up"
  (itemClick)="onSpeedDialItemClick($event)"
></sui-speed-dial>

<sui-speed-dial 
  [items]="stateItems"

  direction="down"

Installation

npm install @synerity/ui

Basic Usage

The SpeedDial component provides a floating action button that expands to show additional action items. 

<sui-speed-dial 
  [items]="speedDialItems"
  buttonIcon="⚡"
  direction="up"
  (itemClick)="onItemClick($event)"
></sui-speed-dial>

Items Configuration

Each item in the SpeedDial can have a label, icon, command function, and disabled state: 

speedDialItems = [
  { 
    label: 'Add', 
    icon: '➕', 
    command: () => this.addItem() 
  },
  { 
    label: 'Edit', 
    icon: '✏️', 
    command: () => this.editItem() 
  },
  { 
    label: 'Delete', 
    icon: '🗑️', 
    command: () => this.deleteItem(),
    disabled: true 
  }
];

Directions

SpeedDial can expand in four different directions:

  • up: Items expand upward (default)
  • down: Items expand downward
  • left: Items expand to the left
  • right: Items expand to the right

Props

Property Type Default Description
items Array<SpeedDialItem> [] Array of action items to display
direction 'up' | 'down' | 'left' | 'right' 'up' Direction in which items expand
buttonIcon string '+' Icon for the main button
buttonLabel string '' Label for the main button
disabled boolean false Whether the SpeedDial is disabled

Events

Event Type Description
itemClick SpeedDialItem Emitted when an item is clicked

SpeedDialItem Interface

Property Type Description
label string Optional label for the item
icon string Optional icon for the item
command () => void Optional function to execute when clicked
disabled boolean Whether the item is disabled

Accessibility

  • Keyboard navigation support with Tab and Enter keys
  • ARIA attributes for screen readers
  • Focus management when expanding/collapsing
  • Proper button semantics for all interactive elements
  • Disabled state support for individual items
