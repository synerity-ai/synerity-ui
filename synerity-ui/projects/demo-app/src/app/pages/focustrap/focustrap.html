Focus Trap

A component that traps focus within a specific element, commonly used in modals and dialogs for accessibility.

Basic Focus Trap

This container traps focus. Use Tab and Shift+Tab to navigate between elements. Focus will cycle within the container.

Focus Trap Demo

Try using Tab to navigate - focus will stay within this container.

 
<sui-focus-trap 
  [enabled]="true"
  [autoFocus]="true"
  class="focus-trap-demo"
>
  <div class="bg-white p-6 rounded-lg border-2 border-blue-500 space-y-4">
    <input type="text" placeholder="First input">
    <input type="text" placeholder="Second input">
    <button>First Button</button>
    <button>Second Button</button>
  </div>
</sui-focus-trap>

Modal Example

Click the button below to open a modal with focus trap. Focus will be trapped within the modal.

Contact Form

Fill out the form below. Focus is trapped within this modal.

 
<div *ngIf="isModalOpen" class="modal-overlay">
  <sui-focus-trap [enabled]="true" [autoFocus]="true">
    <div class="modal-content">
      <h3>Contact Form</h3>
      <form (ngSubmit)="onFormSubmit()">
        <input type="text" [(ngModel)]="formData.name">
        <input type="email" [(ngModel)]="formData.email">
        <textarea [(ngModel)]="formData.message"></textarea>
        <button type="submit">Submit</button>
        <button type="button" (click)="closeModal()">Cancel</button>
      </form>
    </div>
  </sui-focus-trap>
</div>

How to Use Focus Trap

Installation

Import the FocusTrap component from the UI library:

import { FocusTrap } from 'ui-lib';

// In your component imports
imports: [FocusTrap]

Basic Usage

Wrap any content with the focus trap component:

<sui-focus-trap>
  <div>
    <button>Button 1</button>
    <button>Button 2</button>
    <input type="text">
  </div>
</sui-focus-trap>

Component Properties

  • enabled: Boolean to enable/disable focus trapping
  • autoFocus: Boolean to automatically focus the first element
  • style: Custom CSS styles
  • styleClass: Custom CSS class

Common Use Cases

  • Modals: Trap focus within modal dialogs
  • Dropdowns: Keep focus within dropdown menus
  • Popovers: Trap focus in popover content
  • Forms: Keep focus within complex forms
  • Navigation: Trap focus in mobile navigation menus

Best Practices

  • Modal Dialogs: Always use focus trap in modals for accessibility
  • Auto Focus: Enable auto focus for better user experience
  • Escape Key: Implement escape key handling to close modals
  • Focus Restoration: Restore focus to trigger element when closing
  • Testing: Test with keyboard navigation to ensure proper behavior

Accessibility Features

  • Tab Navigation: Cycles through focusable elements
  • Shift+Tab: Reverse navigation support
  • Focus Indicators: Maintains visible focus indicators
  • Screen Readers: Compatible with screen reader navigation

Focusable Elements

The component automatically detects these focusable elements:

  • button elements
  • input elements
  • select elements
  • textarea elements
  • Elements with href attribute
  • Elements with tabindex (except -1)

API Documentation

Properties

Property Type Default Description
enabled boolean true Enable or disable focus trapping
autoFocus boolean true Automatically focus the first element
style any {} Custom CSS styles
styleClass string '' Custom CSS class

Methods

  • getFocusTrapClass(): Returns CSS classes for the focus trap
  • getFocusTrapStyle(): Returns inline styles for the focus trap
  • isEnabled(): Checks if focus trap is enabled

Accessibility

  • ARIA: Maintains ARIA attributes of wrapped elements
  • Keyboard Navigation: Full Tab and Shift+Tab support
  • Screen Reader: Compatible with screen reader navigation
  • Focus Management: Proper focus cycling and restoration

Styling

The component uses CSS classes that can be customized:

  • .sui-focus-trap: Main container
