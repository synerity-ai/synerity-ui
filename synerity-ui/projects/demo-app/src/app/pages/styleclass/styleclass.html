Style Class

A component for dynamically adding, removing, and toggling CSS classes on elements.

Basic Style Class

Click the buttons below to add, remove, or toggle CSS classes on the target element.

Target Element

This element will have CSS classes added, removed, or toggled based on the buttons above.

<sui-style-class 
  selector="#demo-element"
  enterClass="highlight"
  leaveClass="fade-out"
  toggleClass="active"
>
  <button (click)="addHighlight()">Add Highlight</button>
</sui-style-class>

Toggle Classes

Use toggle functionality to switch between different states.

Toggle Target

This element can have multiple classes toggled independently.

<sui-style-class 
  selector="#toggle-element"
  toggleClass="highlighted"
>
  <button (click)="toggleHighlight()">Toggle Highlight</button>
</sui-style-class>

Custom Configuration

Style class with custom target selector and multiple class operations.

Custom Target

This element uses custom CSS classes for styling and animations.

<sui-style-class 
  selector=".custom-target"
  enterClass="custom-enter"
  leaveClass="custom-leave"
  toggleClass="custom-toggle"
  styleClass="custom-style-class"
>
  <button (click)="customOperation()">Custom Operation</button>
</sui-style-class>

How to Use Style Class

Installation

Import the StyleClass component from the UI library:

import { StyleClass } from 'ui-lib';

// In your component imports
imports: [StyleClass]

Basic Usage

Wrap any element with the style class component:

<sui-style-class selector="#my-element" toggleClass="active">
  <button (click)="toggle()">Toggle Class</button>
</sui-style-class>

Component Properties

  • selector: CSS selector for the target element
  • enterClass: Class to add when entering
  • leaveClass: Class to add when leaving
  • toggleClass: Class to toggle on/off
  • target: Direct element reference (alternative to selector)
  • style: Custom CSS styles
  • styleClass: Custom CSS class for the component

Common Use Cases

  • Animations: Add/remove animation classes
  • State Management: Toggle active/inactive states
  • Theme Switching: Toggle dark/light themes
  • UI Feedback: Add highlight or error states
  • Responsive Design: Toggle mobile/desktop layouts

Best Practices

  • CSS Transitions: Use CSS transitions for smooth animations
  • Semantic Classes: Use meaningful class names
  • Performance: Avoid frequent class changes on large elements
  • Accessibility: Ensure class changes don't break accessibility
  • Consistency: Use consistent naming conventions

CSS Integration

Define your CSS classes to work with the component:

.highlight {
  background-color: #fef3c7;
  border-color: #f59e0b;
  transform: scale(1.05);
  transition: all 0.3s ease;
}

.active {
  opacity: 1;
  visibility: visible;
}

API Documentation

Properties

Property Type Default Description
selector string 'body' CSS selector for target element
enterClass string '' Class to add when entering
leaveClass string '' Class to add when leaving
toggleClass string '' Class to toggle on/off
target ElementRef | string 'body' Direct element reference
style any {} Custom CSS styles
styleClass string '' Custom CSS class

Methods

  • addClass(className): Add a CSS class to the target element
  • removeClass(className): Remove a CSS class from the target element
  • toggleClassMethod(className): Toggle a CSS class on the target element
  • enter(): Add the enter class
  • leave(): Add the leave class
  • toggle(): Toggle the toggle class
  • getStyleClassClass(): Returns CSS classes for the component
  • getStyleClassStyle(): Returns inline styles for the component

Accessibility

  • ARIA: Maintains ARIA attributes of target elements
  • Focus Management: Does not interfere with focus
  • Screen Reader: Compatible with screen reader announcements
  • Keyboard Navigation: Works with keyboard interactions

Styling

The component uses CSS classes that can be customized:

  • .sui-style-class: Main container
